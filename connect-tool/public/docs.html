Quickstart

From zero to a connected account in five steps.

1

Receive your API key

We issue one cmlk_live_* key per workspace during onboarding (and an optional cmlk_test_* key for sandbox flows that never fire webhooks). The key is the only credential your backend needs.

2

Mint an SDK token (backend)

Whenever an end-user is about to connect, mint a short-lived JWT that binds them to your workspace. The token expires in 30 minutes. 

curl -X POST https://smconnector.camaleonicanalytics.com/v1/sdk-tokens \
  -H "Authorization: Bearer cmlk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"user_id":"your-end-user-id","ttl":1800}'
# → { "sdk_token": "eyJhbGciOi…", "expires_at": "…" }
3

Embed the SDK (frontend)

Drop the script tag in, or install from npm. The popup opens from a real user click — no popup-blocker drama.

<script> tag
npm
<script src="https://smconnector.camaleonicanalytics.com/connect-sdk.js"></script>
<script>
  const handle = CamaleonicConnect.init({
    sdkToken: SDK_TOKEN_FROM_YOUR_BACKEND,
    workspace: "your-slug",
    platforms: ["twitch", "instagram"],
    onSuccess: ({ accountIds, platform }) => persist(accountIds, platform),
    onError:   (err)  => console.warn(err),
    onExit:    ()     => console.log("user closed"),
  });
  button.onclick = () => handle.open("twitch");
</script>
npm i @camaleonic/connect
import CamaleonicConnect from '@camaleonic/connect';

const handle = CamaleonicConnect.init({
  sdkToken, workspace: 'your-slug',
  platforms: ['twitch', 'instagram'],

4

Read normalized data

Once onSuccess fires you have an accountId. Read profile, audience, content, engagement, and more — same shape across every platform. 

curl https://smconnector.camaleonicanalytics.com/v1/accounts/<id>/identity \
  -H "Authorization: Bearer cmlk_live_xxx"
# → { "platform": "twitch", "platform_user_id": "501116841",
#     "username": "alex_cg_11", "followers_count": 1234, … }
5

Subscribe to webhooks (optional)

Get notified when accounts connect or disconnect, or when a token fails to refresh. Signed with HMAC-SHA256. 

curl -X POST https://smconnector.camaleonicanalytics.com/v1/webhook-endpoints \
  -H "Authorization: Bearer cmlk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://yourapp.com/hooks/camaleonic","events":["account.connected","account.disconnected","token.refresh_failed"]}'

Methods

  • handle.open(platform?) — open the popup. Pass a platform key (e.g. "twitch") to skip the chooser. Must be called from a real user gesture (click handler) to dodge popup-blockers.
  • handle.close() — force-close the popup.

Webhook signing

Every delivery carries an X-Camaleonic-Signature header of the form t=<unix-seconds>,v1=<hex-hmac>. Verify by recomputing the HMAC over "<t>.<raw-body>" with the secret you got from POST /v1/webhook-endpoints.

Use the raw request body, not JSON.stringify(parsed). Re-serialising introduces ordering and whitespace differences that break the HMAC.
Node.js
Python
import { createHmac, timingSafeEqual } from 'node:crypto';

export function verify(req, secret) {
  const sigHeader = req.headers['x-camaleonic-signature'];
  if (!sigHeader) return false;
  const parts = Object.fromEntries(
    sigHeader.split(',').map((kv) => kv.split('='))
  );
  const t = parts.t;
  const v1 = parts.v1;
  if (!t || !v1) return false;
  // Reject deliveries older than 5 minutes (replay protection).
  if (Math.abs(Date.now() / 1000 - Number(t)) > 300) return false;
  const expected = createHmac('sha256', secret)
    .update(`${t}.${req.rawBody}`)
    .digest('hex');
  const a = Buffer.from(expected, 'hex');
  const b = Buffer.from(v1, 'hex');
  return a.length === b.length && timingSafeEqual(a, b);
}
import hmac, hashlib, time

def verify(headers, raw_body, secret):
    sig_header = headers.get('X-Camaleonic-Signature', '')
    parts = dict(kv.split('=', 1) for kv in sig_header.split(','))
    t, v1 = parts.get('t'), parts.get('v1')
    if not t or not v1:
        return False
    if abs(time.time() - int(t)) > 300:
        return False
    expected = hmac.new(
        secret.encode(),
        f"{t}.{raw_body.decode()}".encode(),
        hashlib.sha256,
    ).hexdigest()
    return hmac.compare_digest(expected, v1)

Events

EventFires when
account.connectedAn end-user finishes the OAuth flow and the seed succeeds.
account.disconnectedDELETE /v1/accounts/:id succeeds.
account.refreshedToken refresh worker swapped a fresh access token in.
token.refresh_failedPlatform rejected our refresh attempt — the end-user needs to reconnect.
token.expiredStored token reached its issuer-side expiry without a refresh.

Retry policy

Failed deliveries retry on the following backoff: 1m → 5m → 30m → 2h → 12h → 24h (6 attempts total). After that the delivery is marked abandoned and stops retrying. Inspect deliveries via GET /v1/webhook-deliveries.

Authentication

Two credential types, two audiences. Never let the client browser see a cmlk_* key — it can read every account in your workspace.

CredentialWhereUse
cmlk_live_*Your backendAll /v1/* calls. Long-lived. Hashed at rest, plaintext shown ONCE on issue.
cmlk_test_*Your backendSandbox. Same surface, accounts marked is_test, webhooks not fired.
sdk_token (JWT)BrowserMinted per end-user via POST /v1/sdk-tokens. ≤30 min lifetime.
